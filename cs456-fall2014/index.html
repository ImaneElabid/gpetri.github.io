Course Overview

This course is based on the draft book “Programming Languages: Build, Prove, and Compare” by Normal Ramsey. A fundamental principle of the book and the course, is to present programming languages concepts by firstly providing a small language that incorporates each new fundamental concept, and secondly presenting an interpreter with which one can experiment with the new concept. Experimenting with interpreters implies both, writing programs which we run using the interpreter, as well as modifying the interpreter itself to add or modify features of the language. In doing so, the student will learn about the fundamental tools for programming language design and formalization, namely: syntax, semantics and the implementation of simple interpreters; as well as some of the fundamental concepts of programming languages, including: higher-order functions, types, objects and automatic memory management among others.

This course will not be a survey of existing languages nor an extensive exploration of all language features. We will concentrate of some of the most important features, which will be gradually introduced through micro-languages, allowing the student to appreciate and internalize their importance by using and modifying these features, and comparing the different languages.

Some of the questions we will address in this course are:

  1. Which are some of the mainstream models of computation, and which are the language features that enable them? To answer this we will study the fundamental principles behind imperative, functional, object oriented, and time permitting logical programming languages.
  2. What is the meaning of a given programming language feature, and how is it formalized and implemented? What is the meaning of a program written in a certain programming language? To answer this question we will use operational semantics, a fundamental tool in the design and implementation of programming languages.
  3. What are some important properties of programs and programming languages, and how can we mathematically enforce them and prove them? While this course emphasis is on the design and implementation of some of the fundamental concepts in programming languages, we will in some cases consider important desirable properties of languages, and how can we mathematically guarantee them.
  4. How can we extend a certain programming languages to include new features? How can we compare different programming languages with different feature sets? We will present several languages, and interpreters, and we will consider programming and extending the language across these languages to be able to compare them, and evaluate their features.
  5. Finally, an expectation of this course is that by being exposed to different language features, paradigms, disciplines, and their implementations, students will better understand their fundamental tool, and hence become better programmers.

The course will be roughly divided in five sections plus an optional one:

  1. Imperative Programming: This part of the course will present a tiny imperative programming language. This language will both be used to introduce imperative programming itself, and the tools that will be used throughout the course to present the notions of programming languages, namely: abstract syntax, operational semantics, and how to generate an interpreter from an operational semantics.
  2. Functional Programming: This is a large part of the course in which we will study the fundamental notions that make functional programming practical and attractive. These include higher-order functions, recursion and lists.
  3. Automatic Memory Management: In this part we will consider the implementation of a garbage collector of the previous part.
  4. Types: In this part of the course we will introduce the unifying notion of types, and how it manifests in different languages of interest. We will see how types can make programs safer, and more elegant.
  5. Object Oriented Programming: We will consider the notions of information hiding and abstraction, which are paramount to the scalability and maintainability of large software systems.
  6. Logical Programming (optional): This more arcane programming model is based around the notions of logics and provability. While less popular than the topics considered before, it has a strong impact in the area of mechanized mathematics.

For each of these parts we will present one or more incarnations of the concepts considered in a micro language, and we will consider programming with such micro languages, as well as extensions to these.

Prerequisites

It is assumed that students taking this class would have had some exposure in compiler construction (such as CS 352) and C programming. While functional programming is not a requirement, students not familiar with functional programming are encouraged to study functional programming from the beginning of the course. Some resources are available at the end of this note.

Programming Assignments

The mantra of this course is: learn by doing. There will be assignments given every week. Many assignments will be taken from the book exercises, and it will envolve using and extending the interpreters presented in class. These assignments will be individual, but collaboration is encouraged.

Project

There will be a semester-long project that will involve extending one of the languages considered in the course with a feature not introduced in the course, or writing a micro-version of a language not covered in the course. The details of this project assignment will be announced half-way through the semester.
